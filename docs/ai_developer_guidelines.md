# AI Developer Guidelines: Campus Project

> **IMPORTANT:** This document serves as the absolute source of truth for any developer or AI Agent working on the `campus-project`. Before proposing code changes, you must read and adhere to these guidelines to maintain architectural consistency.

## 1. Technology Stack
This project uses a bleeding-edge, highly optimized stack.

*   **Framework System:** Next.js 16.1.6 (App Router exclusively).
*   **UI Library:** React 19.2.3. Emphasize Server Components (`async function`) and React 19 hooks (`useActionState`, `useFormStatus`, `useOptimistic`).
*   **Styling Engine:** Tailwind CSS v4. Do not use legacy v3 utility patterns or external CSS modules unless absolutely necessary.
*   **Animation Engine:** Framer Motion. Reserved for high-impact micro-animations (Stories, modals, skeleton loaders). Keep the UI feeling native and premium.
*   **Database & Auth:** Supabase (PostgreSQL + Auth + Storage). Uses `@supabase/ssr` (v0.8.0) and `@supabase/supabase-js`.
*   **Icons:** Lucide React.
*   **Language:** Strict TypeScript.

## 2. Directory Architecture & Conventions

*   **`src/app/`**: Contains the Next.js routing. 
    *   **Rule:** Pages here should predominantly be Server Components fetching data from Supabase directly, passing initial data down to Client Components.
*   **`src/components/`**: Reusable UI blocks.
    *   **Rule:** Prefer composition and atomic design (e.g., `StoryViewer.tsx`, `FeedCard.tsx`).
*   **`src/lib/`**: Contains utility functions and API clients.
    *   **Rule:** Use this for centralized Supabase clients (`server.ts`, `client.ts`).
*   **`schema.sql`**: The single source of truth for the database structure. Any changes to the DB logic must be reflected here first.

## 3. Supabase Integration Rules

### 3.1. Authentication & Profiles
*   User profiles are automatically generated via the PostgreSQL trigger `handle_new_user()` when an account is created in Supabase Auth.
*   Do **not** perform frontend inserts to `profiles` on signup. Only update the profile once the user reaches the `/onboarding` flow.

### 3.2. Data Fetching & Mutations
*   **Server Actions:** All data mutations (likes, posting, commenting) must use React 19 Server Actions. Do not use traditional API routes (`/api/...`) for basic DB operations.
*   **Realtime Subscriptions:** For real-time functionality (e.g., chat messages, live feed updates), use Supabase channel subscriptions (`supabase.channel().on(...)`) localized cleanly inside Client component `useEffect` hooks.

### 3.3. File Storage
*   The project uses Supabase Storage with buckets: `posts`, `avatars`, `stories`.
*   Always ensure you respect the RLS policies attached to these buckets when uploading or retrieving URLs.

## 4. UI/UX Principles
*   **Dynamic and Alive:** Incorporate hover states, smooth transitions, and loading skeletons. No action should feel instant without visual feedback.
*   **Mobile-First Design:** Since the primary user base consists of university students on their phones, all Tailwind classes must be optimized for mobile (`w-full`, `max-w-md` for centered app layouts) before scaling up with `md:` prefixes.
*   **Glassmorphism & Gradients:** The app employs a premium, vibrant aesthetic. Leverage Tailwind's backdrop-blur and modern styling presets.
